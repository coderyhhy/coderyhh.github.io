JavaScript

原型相关

根据程序员约定俗成的规矩，首字母大写的一般都是构造函数，但是也有不是的：Map()、Set()、Math()

构造函数的用处：用于创建一个属于当前类型的实例对象。

一个函数是否是构造函数，只能通过同步函数调用方式进行区分，在函数声明的时候是无法区分的。

在 JS 中内存分为几部分

一共分为两部分：

  • 栈内存
    • 只能存放基本数据类型，地址值。
  • 堆内存
    • 可以存储对象，可以存放基本数据类型

注意：变量和函数名是一个东西，统称为标识。

原型相关

  1. 是什么？

    • 所有的实例对象都具有隐式原型 （_proto_ ）
    • 所有函数都有显示原型（prototype）
    • 实例对象的隐式原型只想构造函数的显示原型
    • 所有的原型对象都具有 constructor 属性，指向对应的构造函数。

    总结：一个原型对象，两个原型属性。

  2. 为什么要用？

    • 直接目的：可以让原型对象身上的数据，共享给所有实例对象使用。
    • 最终目的：可以将本来每个实例对象身上相同的方法，全部提取到组合成一个函数，放在原型对象上，用于节省内存空间。

  3. 怎么用？

    • 将数据放在原型对象身上，实例对象如果使用该属性，自身如果没有就会顺着原型链找到该原型对象，从它身上获取该属性。
    • 读取属性就会顺着原型链向上查找，设置属性不会顺着原型链向上查找，会直接复制给原型。

  4. 在哪儿用过？

    • 在 Vue 全局事件总线

每次执行依次匿名函数声明，就会创建一个全新的匿名函数。

new 做的事情

  • 自动声明形参变量，并将传入的实参数据赋值给对应的形参。
  • 自动声明 this 变量，并传入一个全新的实例对象。
  • 将当前构造函数的显示原型属性值，赋值给实例对象的隐式原型属性。
  • 默认 return 当前实例对象

在面试过程中遇到原型打印题，需要注意：到底修改了原型对象身上的属性，还是换了原型对象

什么时候换的原型对象

  • 在 new 之前，还是 new 之后

给基本数据类型添加属性是不会报错的，会出现一个包装类对象进行帮忙

  • Number –> Numbet() 包装类
  • String –> String() 包装类
  • Boolean –> Boolean() 包装类

如果给基本数据类型添加 / 读取属性，就会创建一个对应的包装类对象，该对象会在该行代码执行结束之后，自动销毁，不会影响到当前的代码。

如果，现在有一个变量 data，内部会存放数据，需要实现一个函数，返回该函数的数据类型

  • 解决：

    • typeof
      • 语法：typeof data
      • 返回值的数据类型是字符串
      • 返回值
        • ‘number’ ‘string’ ‘boolean’ ‘undefined’ ‘object’ ‘function’ ‘symbol’ ‘bigint’
      • 优点：
        • 将需要辨别的数据交给它，它可以快速明了的返回数据类型
      • 缺点：
        • 无法清除的区分特色的对象数据类型，例如 object 和 array
        • 会将null 错误识别为 object
          • 因为在 JS 底层，数据都是 2 进制存储
          • null 和 object 前三位都是 000， 所以被误识别
    • instanceof
      • 语法： a instanceof
      • 返回值的数据类型是布尔值
      • 返回值
        • true false
      • 用于判断 a 的实例对象是否为 b 数据类型
      • 优点：
        • 判断结果很准确
      • 缺点；
        • 它只能判断 a 是不是 b 类型，无法判断 a 具体是什么类型。
    • 借用 Object.toString 的方法
      • 为什么不使用 Array 的 toString， 因为数组的 toString 经过了重写，会将数组的中括号去除，并将内部所有的数据，以逗号进行拼接，成了字符串
      • 同理等同于，查看原型对象的 constructor 一样

  • 语法糖：

    • 它是新的语法，但是底层其实还是之前的旧语法
    • {} –> new Object
    • [] –> new Array
    • function() {} –> new Function

终极原型链相关

万物皆对象,除了基本数据类型以外

所有的对象都是 Object 的实例对象

所有的函数都是 Function 的实例对象

  • 包括它自己
  • 在 JS 中只有 Function 才能创建函数

在 JS中, 用 C语言实现的东西:Object Function Function.prototype 这三个

在 JS 中, 强行修改了四条线

  1. Function._propop_ === Function.prototype
    • 身份证明
      • 如果没有这条线,那么 Function就不在是函数了
    • 使用方法
      • 所有函数共享的方法都会放在 Function.ptototype 上, 如果没有这条线, 那么 Function 将无法使用 call bind apply 等等方法
  2. Object._proto_ === Function.prototype
  • 同上
  1. Function._proto_ === Object.ptoyotype
    • 如果没有这条线, 那么原型链将会出现两个终点, 那么函数就不在是对象了
  2. Object._proto_ === null
    • 如果没有这条线, 那么原型链将会进入死循环, 无论是那个实例对象查找属性, 都会在 Object 卡住

什么是伪数组

  1. 使用下标作为属性名
  2. 具有 length 属性
  3. 伪数组不能用数组方法
    • 伪数组的原型链上没有出现 Array 的原型对象

作用声明域

在编程界, 作用域一共分为两种

  1. 词法作用域 (静态作用域)
    • 函数的作用域声明位置有关, 与调用位置无关, JS 就是静态作用域
  2. 动态作用域
    • 函数的作用域与声明位置无关, 与调用位置有关

在 JS 中, 只有词法作用域, 没有动态作用域

作用域就是用来约束变量的作用范围, 作用域其实就是变量的查找规则

预解析规则(变量提升)

  1. 变量提升
    • 变量提升是局部提升, 它会将标识的声明提升到最顶端, 标识的赋值会留在原地
  2. 函数提升
    • 函数提升是整体提升, 声明和赋值都会提升到最顶端

注意: 区分清楚标识声明和标识赋值

例如 var a = 1

  • 标识声明 var a
  • 标识赋值 a = 1

函数表达式提升的是变量提升, 只会将标识声明提前, 赋值将会留在原地

预解析的优先级: 函数优先, 函数提升优先于变量提升

预解析的范围:

  • 会将标识声明提升到当前作用域的最顶端, 不会穿透作用域

注意: 变量查找都是从内向外查找

作用域分类

  1. 全局作用域
    • 它是 JS 最大的顶级作用域
    • 所有人都可以获取到他身上的数据
    • 扩展: 全局作用域其实也是函数作用域, 因为 JS 引擎会将主线程代码放在一个匿名函数中调用执行
    • 但是它是一个最顶级的函数, 所以所有人都可以用
  2. 局部作用域 (函数作用域)
    • 与函数有关, 每个函数都有自己独立的一片作用域
    • 内部的数据不能被外侧访问
  3. 块级作用域
    • 只要使用 const let 就会出现 ES6 新增的块级作用域
    • 注意点: let 和 const 需要根据外层的大括号进行配合, 生成块级作用域
    • 在同一个大括号中, 多次使用 let const 都只会生成一个块级作用域
  4. 形参默认值( es6新增语法)
    • 只要函数使用了形参默认值, 那么函数内部的代码就会在新生成的块级作用域中执行
      • 注意: 函数内部在块级作用域中, 函数形参还在函数作用域中
      • 函数被调用, 如果函数内部和函数形参的名字相同, 那么他们的初始值也会相同, 函数形参会将自己的值传递一份给函数内部同名变量
    • 作用域是一套规则, 一套定义(并不是真正意义存在的)
      • 变量对象就是作用域实现的结果

闭包

  1. 闭包的产生条件
    • 闭包是一种引用关系
      • 必须具有内外函数嵌套
      • 内部函数必须使用外部函数的变量
    • 闭包分为两种
      • 有效闭包: 外部函数被调用的时候, 会将内部函数返回出去, 那么这个闭包就可以持续保留, 称为有效闭包
      • 无效闭包: 外部函数被调用的时候, 没有将内部函数返回出去, 那么这个闭包就会快速销毁, 称为无效闭包
