Canvas 和 SVG 的区别是什么？

区分题：答题思路为：先说一，再说二，最后说不同点。

  1. Canavs 主要是用笔刷来绘制 2D 图形的。
  2. SVG 主要是用标签来绘制不同规则矢量图的。
  3. 相同点：都是主要用来画 2D 图形的。
  4. 不同点 Canvas 画的是位图， SVG 画的是矢量图。
  5. 不同点 SVG 节点过多时渲染慢，Canvas 性能更好一些，但写起来更复杂。
  6. 不同点 SVG 支持分层和事件，Canvas 不支持，但是可以用库实现。

两种盒模型（box-sizing）的区别

区分题：答题思路为：先说一、再说二、最后说不同点

第一种盒模型是 content-box，即width 指定的是 content 区域的宽度，而不是实际的宽度，公式为：

实际宽度 = width + padding + border

第二种盒模型是 border-box，即 width 指定的是左右边框外的距离，公式为：

实际宽度 = width

相同点：都是用来指定宽度的。

不同点：border-box 更好用

原型链是什么？

大概念题：答题思路为大概化成小概念（分割），抽象化成具体（举例）

假设我们有一个人普通的对象 x={} ， 这个 x 会有一个隐藏属性， 叫做 __propt__， 这个属性会指向 Object.prototype。

x._proto_ === Object.prototype // 原型。

此时，我们说的 x 的原型就是 Object。prototype，或者说 Object.prototype 是 X 的原型。

而这个 _proto_ 属性的唯一作用就是用来指向 X 的原型的。

如果没有 _proto_ 属性，x 就不知道自己的原型是谁了。

接下来我们说说原型链

假设有一个数组对象 a=[]，这个 a 也会有一个隐藏属性，叫做__proto__，这个属性会指向 Array.prototype

a._proto_ = Array.prototype

此时，我们说 a 的原型是 Array.prototype，跟上面的 x 一样， 但又有一点不一样， 那就是 Array.prototype也有一个隐藏属性_proto_ 指向Object.prototype

用 x 表示 Array.prototype

// Array.prototype === Object.prototype

x._proto_ === Object.prototype

这样一来，a 就有两层原型：

  1. a 的原型是 Array.prototype
  2. a 的原型的原型就是 Object.prototype

于是就通过隐藏属性 _proto_ 形成了一个链条

a ==> Array.prototype ==> Object.prototype

这就是原型链。

怎么做：

看起来只要改写 x 的隐藏属性 _proto_ 就可以改变 x 的原型（链）

x._proto_ = 原型

但是这不是标准推荐的写法，为了设置 x._proto_ 推荐的写法是

const x = Object.create

// 或者

const x = new 构造函数() // 会导致 x._proto_ === 构造函数.prototype

解决了什么问题：

在没有 class 的情况下实现 【继承】。以 a ===> Array.prototype ===> Object.prototype 为例。

  1. a 是 Array 的实例，a 拥有 Array.prototype 里的属性。
  2. Array 继承了 Object （注意专业术语的使用）。
  3. a 是 Object 的间接实例，a 拥有 Object. prototype 里的属性

这样一来 a 就即拥有 Array.prototype 里的属性，又拥有 Object.prototype 里的属性。

优点：

  • 简单，优雅

缺点：

  • 跟 class 相比， 不支持私有属性

怎么解决缺点：

  • 使用 class，那是 class 是 ES6引入的。
